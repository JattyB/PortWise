------------------------------------------------------------
WHY THIS MATTERS
------------------------------------------------------------
The TLS engine tests protocol VERSIONS (1.0-1.3) but never enumerates CIPHER
SUITES. Real TLS findings in a pentest report are about ciphers: RC4, 3DES,
CBC-mode, NULL, EXPORT, anonymous DH, no forward secrecy, weak DH/key size,
SHA1 signatures. PortWise currently cannot report any of these. This closes
that gap.

------------------------------------------------------------
WHAT TO BUILD
------------------------------------------------------------

STEP 1 — Prefer NSE ssl-enum-ciphers (from PT1).
In a new file portwise/modules/tls/cipher_checks.py:
  - If nse_ssl_ciphers(service) (from PT1) has data, USE IT. NSE's
    ssl-enum-ciphers already grades ciphers (A/B/C/F) per TLS version and lists
    key exchange info + warnings. Parse that structured data into findings.

STEP 2 — Native fallback cipher probe (no NSE, no external tools).
  - If NSE data is absent, do a native enumeration using Python ssl:
    iterate a curated list of cipher categories and attempt handshakes with
    restricted cipher strings via SSLContext.set_ciphers() to detect whether
    weak families are accepted:
       * RC4 (set_ciphers("RC4"))
       * 3DES (set_ciphers("3DES"))
       * CBC legacy (set_ciphers("AES128-SHA:AES256-SHA:DES-CBC3-SHA"))
       * NULL/EXPORT/anon (set_ciphers("aNULL:eNULL:EXPORT"))
       * No-PFS (set_ciphers("kRSA") — non-ephemeral RSA kx = no forward secrecy)
    For each: try a handshake limited to that family; success = "supported".
    Wrap each in try/except (modern OpenSSL may refuse to even build the
    context for some — treat as "not testable", record once, no per-host noise
    a la fixpack #6).
  - Capture negotiated cipher details from a normal handshake:
    cipher name, protocol, secret bits (sock.cipher()).

STEP 3 — Findings (tie to fix #1 categories).
  - "Weak TLS Cipher Suites Supported" (RC4/3DES/CBC) -> MEDIUM, vulnerability,
    list the specific suites in evidence.
  - "Insecure TLS Cipher Suites Supported" (NULL/EXPORT/anon) -> HIGH, vulnerability.
  - "No Forward Secrecy" (kRSA accepted, no ECDHE/DHE) -> LOW/MEDIUM, best_practice.
  - "Weak TLS Signature Algorithm" (SHA1 cert sig from PT1 nse_ssl_cert) ->
    MEDIUM, vulnerability.
  - "Weak DH Parameters" if detectable (< 2048-bit) -> MEDIUM (NSE often reports
    dh params; native detection is hard — only report if NSE provides it).
  - Confidence: NSE-backed = LIKELY/CONFIRMED (authoritative grade); native
    handshake success = CONFIRMED (we directly negotiated it).

STEP 4 — Integrate.
  - Call cipher_checks from tls_engine.run() after protocol findings.
  - Respect per-host delay (fixpack #3) so cipher probing doesn't burst.
  - Config: tls.cipher_enumeration (default true), tls.native_cipher_probe
    (default true when NSE absent).
